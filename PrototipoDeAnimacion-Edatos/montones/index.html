Explicación General del Algoritmo

Un montón (heap) es una estructura de datos en forma de árbol binario completo, en la que los elementos se organizan según una propiedad específica que depende del tipo de montón:

  • Montón Mínimo (Min-Heap): En cada nodo, el valor es menor o igual al de sus hijos, lo que garantiza que el valor más bajo se encuentre siempre en la raíz.
  • Montón Máximo (Max-Heap): En cada nodo, el valor es mayor o igual al de sus hijos, lo que asegura que el valor más alto esté en la raíz.

Los montones son útiles porque permiten acceder rápidamente al valor más pequeño o más grande de todos los elementos. Esto los hace efectivos para organizar datos o gestionar prioridades de manera eficiente.

¿En qué consiste?

Un montón mínimo (Min-Heap) funciona de la siguiente manera:

  1. Construcción inicial: Se crea el montón organizando los elementos de forma que el valor más pequeño siempre esté en la raíz. Esto asegura que se cumpla la propiedad de Min-Heap desde el inicio.

  2. Insertar elemento: Cada nuevo elemento se añade en la última posición del árbol y se compara con su nodo padre. Si es menor que su padre, se intercambian, y el elemento sube en el árbol hasta encontrar su posición correcta.

  3. Eliminar el mínimo: El elemento en la raíz (el más pequeño) se elimina, y el último elemento del árbol se mueve a la raíz temporalmente. Después, se ajusta el árbol desplazando el elemento hacia abajo hasta que se restaure la propiedad de Min-Heap.

  4. Reajuste del montón (heapify): Tras una inserción o eliminación, el montón se reorganiza intercambiando los elementos según las relaciones entre padres e hijos, asegurando que el valor más pequeño permanezca en la raíz.

Ejemplo

Montón inicial: []   (vacío).

  1. Insertar elementos:

    Se añade el elemento 5 como raíz del montón.
    - Montón: [5]

    Se añade el elemento 8 como hijo del nodo raíz, sin necesidad de reajustar.
    - Montón: [5]
             /
          [8]

    Se añade el elemento 3. Como 3 es menor que la raíz 5, se intercambian.
    - Montón: [3]
             /   \
          [8]     [5]

    Se añade el elemento 7 como hijo del nodo 8, sin necesidad de reajustar.
    - Montón: [3]
             /   \
          [8]     [5]
          /
        [7]

  2. Eliminar el mínimo (raíz):

    Se elimina el elemento en la raíz, que es 3, y se mueve el último elemento (7) temporalmente a la raíz.
    - Montón temporal: [7]
                      /   \
                    [8]   [5]

    Para restaurar la propiedad de Min-Heap, se intercambia 7 con el menor de sus hijos (5).
    - Montón reajustado: [5]
                        /   \
                      [8]   [7]
