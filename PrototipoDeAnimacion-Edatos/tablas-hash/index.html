TABLAS HASH



Explicación General del Algoritmo

Una tabla hash es una estructura de datos que almacena elementos utilizando una clave única asociada a cada uno de ellos. Esta clave se transforma mediante una función hash en un índice, que indica la posición en la tabla donde se guarda el elemento.

Las tablas hash destacan por su eficiencia al realizar búsquedas, inserciones y eliminaciones, ya que permiten acceder directamente a los elementos mediante su clave. Para manejar situaciones donde dos claves generan el mismo índice (colisiones), se utilizan métodos como listas encadenadas o sondeos para resolver conflictos y mantener el orden.



¿En qué consiste?

Una tabla hash funciona de la siguiente manera:

  1. Inicialización: Se crea una tabla vacía con un tamaño inicial fijo, donde cada posición puede almacenar elementos o referencias a ellos.

  2. Función hash: La función hash toma la clave del elemento y aplica una operación matemática (como el módulo) para transformar la clave en un índice dentro de la tabla. En este ejemplo, utilizamos el módulo del tamaño de la tabla para obtener el índice: clave % tamaño de tabla.

  3. Inserción: El elemento se almacena en la posición de la tabla indicada por el índice generado por la función hash. Si hay una colisión (dos claves generan el mismo índice), se utiliza una técnica de resolución como encadenamiento o sondeo para encontrar un lugar disponible.

  4. Búsqueda: Para buscar un elemento, la clave se transforma en un índice mediante la función hash, y se verifica la posición correspondiente en la tabla. Si hay una colisión, se siguen las referencias para encontrar el elemento deseado.

  5. Eliminación: Para eliminar un elemento, se localiza mediante su índice hash y se elimina de la posición correspondiente. Si hay colisiones, se ajustan las referencias para evitar inconsistencias.



Ejemplo

Tabla hash inicial: [ , , , , ]   (vacía, tamaño 5).

  1. Insertar elementos:

    Se calcula el índice hash para la clave 15.
    - Índice generado: 15 % 5 = 0.
    - El elemento 15 se almacena en la posición 0.
    - Tabla hash: [15, -, -, -, -]

    Se calcula el índice hash para la clave 10.
    - Índice generado: 10 % 5 = 0.
    - Se detecta una colisión en la posición 0. Se utiliza encadenamiento (lista) para almacenar ambos valores.
    - El elemento 10 se añade a la lista en la posición 0.
    - Tabla hash: [[15, 10], -, -, -, -]

    Se calcula el índice hash para la clave 22.
    - Índice generado: 22 % 5 = 2.
    - El elemento 22 se almacena en la posición 2.
    - Tabla hash: [[15, 10], -, [22], -, -]

  2. Búsqueda de elementos:

    Para buscar el elemento 10:
    - Se calcula el índice hash: 10 % 5 = 0.
    - En la posición 0, se recorre la lista y se encuentra el elemento 10.
    - Resultado: Elemento 10 encontrado.

    Para buscar el elemento 22:
    - Se calcula el índice hash: 22 % 5 = 2.
    - En la posición 2, se encuentra el elemento directamente.
    - Resultado: Elemento 22 encontrado.


  3. Eliminar elementos:

    Para eliminar el elemento 10:
    - Se calcula el índice hash: 10 % 5 = 0.
    - En la posición 0, se recorre la lista y se elimina el elemento 10.
    - Tabla hash: [[15], -, [22], -, -]

    Para eliminar el elemento 22:
    - Se calcula el índice hash: 22 % 5 = 2.
    - En la posición 2, se elimina el elemento directamente.
    - Tabla hash: [[15], -, -, -, -]
